# وحدات النظام والمميزات (Modules & Features)

> وثيقة تفصيلية لوحدات النظام التي تغطي جميع المتطلبات.
> مرتبطة بالخماسية الأساسية: **الطلب / البنود / الدفعات / الحركات / الأرشفة**.

---

## 3.1 المستخدمون والصلاحيات والتدقيق

- **الأدوار:** إدارة، مدير إنتاج، مشتريات، مستودع، قص، خياطة، تطريز، جودة، تغليف/كي، تسليم، محاسب تشغيلي.
- **PIN اعتماد** للتابلت (استلام/تسليم/اعتماد إنتاج).
- **سجل تدقيق Audit Log** لكل تغيير مهم: من عدّل؟ متى؟ ماذا؟
- ربط كل إجراء حساس بصلاحية واضحة (لا توجد صلاحيات ضمنية).

## 3.2 العملاء + سياسات الآجل

- **تصنيف العميل:** نقدي / آجل معتمد / متعثر.
- **حد ائتماني** + **مدة سداد**.
- **منع تنفيذ طلب جديد للمتعثر** إلا بموافقة موثقة.
- ربط كل عميل بـ **مندوب مسؤول** + **تقارير ديون حسب المندوب**.

## 3.3 الطلبات (كيان واحد) + المسارات

- **مسارات الطلب:** إنتاج / عينة / تعديل.
- **Parent Order** لربط العينة/التعديل بالطلب الأصل.
- دعم **مقاسات قياسية (S/M/L)** + **تفصيل خاص (قياسات مفصلة)**.

## 3.4 بنود الطلب (Order Items) — مهم جدًا

- “الزي الموحد” ليس بنطال فقط: قميص، بنطال، جاكيت، بدلة، عباءة… إلخ.
- كل بند له: **قماش / لون / تطريز / ملاحظات / صور مرجعية**.

## 3.5 الدفعات + الانتقالات بين الأقسام (بدل الدفاتر)

- إنشاء **دفعات** للقص / الخياطة / التطريز / الجودة / التغليف.
- **تسليم/استلام:**
  - من قسم → إلى قسم.
  - كمية مرسلة / مستلمة.
  - **اعتماد المرسل + اعتماد المستلم (PIN).**
  - أي فرق؟ يسجل **سبب رسمي** (تالف / زيادة / نقص / إعادة عمل…).

## 3.6 فروقات الكميات (Quantity Movements)

أي فرق يصبح **حركة كمية** وليس مجرد ملاحظة:

- زيادة قص (مثال 100 → 120).
- نقص.
- تالف.
- هدر.
- تسليم مجاني.
- إعادة عمل.

> تُنسب لمسؤولية (قسم / شخص) وتظهر في التقارير.

## 3.7 التطريز (وفق رؤية 2026)

- **اعتماد قبل التنفيذ** (Mockup / تصميم).
- **ملف عميل للتطريز:** خيوط / ألوان / غرز.
- **بطاقة مواصفات التطريز PDF**.
- **فصل ماكينة العينات** (Resource مستقل) حتى لا توقف الإنتاج.

## 3.8 المشتريات والمخزون

- **طلب شراء داخلي** مع اعتماد.
- **جدولة:** يومين أسبوعيًا + استثناء طارئ بموافقة.
- **المخزون كسجل حركات** (لا أرصدة مباشرة).
- **لفات القماش + خيوط التطريز** + **حد أدنى / تنبيه**.
- **جرد شهري إلزامي**.

## 3.9 الجودة

- نتائج: **قبول / إعادة عمل / رجوع لقسم سابق / مرفوض (هالك)**.
- **أسباب** + **صور** إلزامية للحالات السلبية.

## 3.10 التسليم والفواتير والصندوق (تشغيلي)

- **سندات تسليم** جزئية / كاملة.
- **فاتورة** مرتبطة بالطلب.
- **مدفوعات + صندوق قبض/صرف**.
- **مطابقة السند قبل الفاتورة** (كما هو معمول به حاليًا).

## 3.11 الأرشفة الإلزامية + إعادة الطلب

- **Checklist إلزامي** لإغلاق الطلب.
- **لا يُغلق الطلب** قبل اكتمال الأرشفة.
- زر **“تكرار طلب”** من الأرشيف (تعديل كميات/مقاسات فقط).

## 3.12 العينات (وفق رؤية 2026)

- **عينة جاهزة من المخزون.**
- **عينة تفصيل بموافقة وتكلفة.**
- **سقف عينات للمندوب.**
- **تقرير تحويل العينات لمبيعات.**
